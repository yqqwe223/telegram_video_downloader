## 🚀 คู่มือการใช้งานด่วน

1. เปิดแอปหรือเว็บไซต์ Telegram แล้วค้นหา **ช่องสาธารณะ/กลุ่มสาธารณะ** ที่มีเนื้อหาที่ต้องการอ้างอิง
2. คลิกที่ข้อความที่ต้องการ → เลือกไอคอน «···» → «คัดลอกลิงก์» เพื่อรับลิงก์เว็บพรีวิว (ตัวอย่าง: `https://t.me/username/123`)
3. วางลิงก์ในช่องป้อนข้อมูลของเครื่องมือนี้ แล้วคลิกปุ่ม "วิเคราะห์"
4. ระบบจะดึงข้อมูลเมตาสาธารณะและแสดงรายการทรัพยากรที่ใช้งานได้
5. เลือกทรัพยากรที่ต้องการ แล้วคลิกขวา → "บันทึกเป็น" เพื่อบันทึกไปยังอุปกรณ์ของคุณ

> 💡 เคล็ดลับการใช้งาน:
> - โปรดตรวจสอบให้แน่ใจว่าช่อง/กลุ่มและเนื้อหานั้นตั้งค่าเป็น "สาธารณะ"
> - หากการวิเคราะห์ล้มเหลว ลองรีเฟรชหน้าหรือตรวจสอบการเชื่อมต่อเครือข่าย
> - สำหรับผู้สนใจเรียนรู้เทคนิค แนะนำให้ใช้ Developer Tools ของเบราว์เซอร์ (F12 → Network → Fetch/XHR) ร่วมด้วย

## ⚠️ ข้อกำหนดการใช้งานและข้อจำกัดความรับผิดชอบ (โปรดอ่านอย่างละเอียด)

โปรเจกต์นี้ยึดถือหลักการ "ความเป็นกลางทางเทคนิค" และ "การปฏิบัติตามกฎหมาย" เป็นพื้นฐาน กรุณาทำความเข้าใจข้อต่อไปนี้ก่อนใช้งาน:

### ✅ พฤติกรรมที่แนะนำ
- วิเคราะห์เฉพาะ **เนื้อหาสาธารณะ** ที่คุณมีสิทธิ์เข้าถึงเท่านั้น
- ใช้ทรัพยากรที่ดึงมาเพื่อ **การเรียนรู้ส่วนตัว / การวิจัย / การใช้ประโยชน์ส่วนตัว** เท่านั้น
- หากต้องการเผยแพร่ต่อ / สร้างงานดัดแปลง / ใช้เชิงพาณิชย์ ต้องขออนุญาตจากเจ้าของลิขสิทธิ์ล่วงหน้า
- ระบุแหล่งที่มาและชื่อผู้สร้างสรรค์ผลงานในชิ้นงานของคุณ เคารพสิทธิ์ของครีเอเตอร์

### ❌ พฤติกรรมที่ต้องห้าม
- พยายามวิเคราะห์เนื้อหาจากช่องส่วนตัว/กลุ่มส่วนตัว/เนื้อหาที่ต้องมีการยืนยันตัวตนหรือจำกัดการเข้าถึง
- ใช้เครื่องมือเพื่อวัตถุประสงค์ในการสคราปเชิงพาณิชย์ / บริการรวบรวมข้อมูล / สร้างรายได้จากโฆษณา
- ส่งคำขอจำนวนมากในเวลาสั้นๆ / ใช้เครื่องมืออัตโนมัติ / โจมตีสร้างภาระให้แพลตฟอร์ม
- ดัดแปลง / ลบ / ปิดบังวอเตอร์มาร์ก / ข้อมูลลิขสิทธิ์ / เมตาดาตาของทรัพยากร
- ใช้เครื่องมือนี้เพื่อเข้าถึง เผยแพร่ หรือกระจายเนื้อหาที่ละเมิดความเป็นส่วนตัว ผิดกฎหมาย หรือละเมิดสิทธิ์

> 📜 หลักการทางกฎหมาย:
> การใช้งานเครื่องมือนี้ต้องเป็นไปตาม "พระราชบัญญัติลิขสิทธิ์" "พระราชบัญญัติว่าด้วยการกระทำความผิดเกี่ยวกับคอมพิวเตอร์" ของประเทศไทย รวมถึง "Terms of Service" และ "Privacy Policy" ของ Telegram

## 🛠 ข้อมูลทางเทคนิคสำหรับผู้พัฒนา (อ้างอิง)

> ผู้ใช้ทั่วไปสามารถข้ามส่วนนี้ได้

### ภาพรวมสถาปัตยกรรม
```
เบราว์เซอร์ผู้ใช้ → โมดูลวิเคราะห์ฝั่งไคลเอนต์ → Telegram Public Web Preview API → การแยกวิเคราะห์ข้อมูลโครงสร้าง → แสดงผลลัพธ์
```

### จุดเด่นทางเทคนิค
- ใช้ `fetch` API ร่วมกับการตั้งค่า CORS Proxy ที่เหมาะสม เพื่อดึงเมตาดาตาจากหน้าเว็บพรีวิวสาธารณะ
- แยกวิเคราะห์ข้อมูลจาก Open Graph Tags (`og:video` / `og:image` / `og:title` ฯลฯ) เพื่อค้นหาลิงก์ทรัพยากร
- ใช้ข้อมูลที่มีโครงสร้าง (JSON-LD / Microdata) จากหน้าเว็บเพื่อเสริมข้อมูลสื่อ
- ใช้การตรวจสอบสองชั้นด้วย Regular Expression + DOM Parsing เพื่อเพิ่มความเสถียรของการตรวจจับลิงก์

### คู่มือการติดตั้งบนเซิร์ฟเวอร์ส่วนตัว (อ้างอิง)
```bash
# 1. โคลนรีพอซิทอรี (ตัวอย่าง)
git clone https://github.com/yourname/telegram-downloader-th.git

# 2. โฮสต์ไฟล์สแตติก (แนะนำใช้ HTTPS)
#    - Vercel / Netlify / Cloudflare Pages (แนะนำสำหรับเริ่มต้น)
#    - Nginx + ใบรับรอง Let's Encrypt (สำหรับเซิร์ฟเวอร์ส่วนตัว)

# 3. ตัวอย่างการตั้งค่า Security Headers (Nginx)
add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline';";
add_header X-Content-Type-Options "nosniff";
add_header Referrer-Policy "strict-origin-when-cross-origin";
add_header X-Frame-Options "DENY";
```

> 🔐 ข้อควรระวังสำหรับการใช้งานจริง:
> - ต้องเปิดใช้งาน HTTPS เพื่อป้องกันการโจมตีแบบ Man-in-the-Middle
> - แนะนำให้ตั้งค่า Rate Limiting เพื่อจำกัดคำขอที่มากเกินไป
> - ควรระมัดระวังในการเปิดเผยลอจิกการวิเคราะห์มากเกินไป เพื่อลดความเสี่ยงในการถูกนำไปใช้ในทางที่ผิด
> - ตรวจสอบและอัปเดตแพ็กเกจ وابستگیอย่างสม่ำเสมอเพื่อปิดช่องโหว่ด้านความปลอดภัย

## ❓ คำถามที่พบบ่อย (FAQ)

**Q: ขึ้นข้อความ "ไม่สามารถดึงเนื้อหาได้" เกิดจากอะไร?**  
A: สาเหตุที่เป็นไปได้: ① ลิงก์ปลายทางเป็นช่องส่วนตัว/กลุ่มส่วนตัว/เนื้อหาที่ต้องยืนยันตัวตน ② เนื้อหาถูกลบหรือตั้งค่าเป็นเฉพาะสมาชิก ③ Telegram ปรับเปลี่ยนโครงสร้างหน้าเว็บพรีวิวชั่วคราว ④ ข้อจำกัดเครือข่าย / ปัญหา CORS วิธีแก้ไข: ตรวจสอบการตั้งค่าสาธารณะ → ลองใช้เครือข่ายอื่น → รอสักครู่แล้วลองใหม่

**Q: วิดีโอ/ภาพที่ดึงมามีน้ำเครื่องหมายการค้า (วอเตอร์มาร์ก) ปกติไหม?**  
A: เครื่องมือนี้ส่งคืนลิงก์ทรัพยากรต้นฉบับที่ Telegram ให้บริการอย่างเป็นทางการเท่านั้น การมีหรือไม่มีวอเตอร์มาร์กขึ้นอยู่กับตั้งค่าตอนอัปโหลดของผู้สร้างเนื้อหา เครื่องมือนี้ไม่มีการเพิ่ม / ลบ / ดัดแปลงใดๆ ทั้งสิ้น

**Q: รองรับของการวิเคราะห์ประวัติข้อความทั้งช่อง/กลุ่มแบบกลุ่มไหม?**  
A: เวอร์ชันปัจจุบันมุ่งเน้นการวิเคราะห์ข้อความสาธารณะแบบเดี่ยว เพื่อให้ความสำคัญกับความเสถียรและการปฏิบัติตามกฎระเบียบ สำหรับคำขอการประมวลผลแบบกลุ่ม กรุณาตรวจสอบความสอดคล้องกับ "Terms of Service" ของ Telegram ด้วยตนเองก่อน

**Q: เครื่องมือนี้เก็บประวัติการใช้งานหรือข้อมูลบัญชี Telegram ของฉันไหม?**  
A: ไม่เก็บครับ โปรเจกต์นี้เป็นหน้าเว็บสแตติกฝั่งไคลเอนต์ล้วน ไม่มีส่วนของแบ็กเอนด์บันทึก 로그 / สคริปต์วิเคราะห์ / ระบบติดตามผ่านคุกกี้ / หรือการเชื่อมต่อบัญชีผู้ใช้ใดๆ การประมวลผลทั้งหมดเกิดขึ้นภายในเบราว์เซอร์ของคุณเท่านั้น และไม่จำเป็นต้องล็อกอิน

**Q: สามารถวิเคราะห์แชทส่วนตัวหรือบทสนทนาระหว่างบุคคลได้ไหม?**  
A: ไม่ได้ครับ เครื่องมือนี้รองรับเฉพาะ **ลิงก์เว็บพรีวิวของช่องสาธารณะ/กลุ่มสาธารณะ** เท่านั้น การวิเคราะห์แชทส่วนตัว บทสนทนาระหว่างบุคคล หรือเนื้อหาที่ต้องล็อกอิน ไม่สามารถทำได้และไม่ควรพยายามทำ นี่คือหลักการพื้นฐานของการเคารพความเป็นส่วนตัวของผู้ใช้และการปฏิบัติตามกฎระเบียบของผลิตภัณฑ์

> เทคโนโลยีไม่มีดีหรือชั่วในตัวเอง สิ่งสำคัญคือ 'เจตนา' และ 'ความรับผิดชอบ' ของผู้ใช้งาน

เราขอเชิญชวนนักพัฒนาและผู้ใช้ทุกท่านร่วมยึดถือค่านิยมต่อไปนี้:

- 🔬 ใช้การเรียนรู้และความอยากรู้อยากเห็นเป็นแรงขับเคลื่อน เพื่อทำความเข้าใจหลักการของเทคโนโลยีเว็บอย่างลึกซึ้ง
- 🤲 เคารพสิทธิ์ของครีเอเตอร์และความเป็นส่วนตัวของผู้ใช้ สร้างนิสัยการระบุแหล่งที่มาและการขออนุญาตอย่างเหมาะสม
- 🌍 มีส่วนร่วมในการส่งเสริมการกระจายเนื้อหาดิจิทัลอย่างสุขภาพดี และการรักษาความหลากหลายทางวัฒนธรรม
- ⚖️ รักษาสมดุลระหว่างการสำรวจทางเทคนิคกับการปฏิบัติตามกฎหมาย ปฏิบัติการพัฒนาอย่างมีความรับผิดชอบ

ร่วมกันสร้างวงจรเชิงบวกของการสร้างสรรค์และการแบ่งปัน ผ่านการใช้งานที่ถูกต้องและมีความรับผิดชอบ ✨
